# Textile.by_parser
Парсер сайта textile.by

Парсер создан с использованием библиотеки Beautiful Soup, для входа на сайт случайным образом выбирает заголовки из списка.
Скрипт настроен на сбор данных каждые 5 часов.
Структура получаемых данных:
- артикул товара;
- URL страницы;
- стоимость;
- производитель;
- тип;
- категория;
- заголовок (наименование товара);
- цвета:
 - название цвета;
 - URL-ссылка на изображение;
- размеры:
 - размеры;
 - общий остаток на складе;
 - остаток на складе для каждого размера;
 - наценки для больших размеров.
- описание.

В базе данных MongoDB были проставлены индексы для наиболее часто используемых полей:
- тип;
- категория;
- стоимость.

Запросы для получения данных из базы данных MongoDB:

- для получения списка всех женских футболок, с удобным для чтения выводом:
db.textile_products.find({"Тип": "Футболки", "Категория": "Женские"}).pretty()

- для получения списка всех женстих футболок в порядке возрастания цены, с удобным для чтения выводом:
db.textile_products.find({"Тип": "Футболки", "Категория": "Женские"}).sort({"Стоимость":1}).pretty()

Создан второй парсер с использованием библиотеки Selenium для работы с динамическим контентом сайта.
Настроен на сбор данных каждые 10 часов.
Структура получаемых данных:
- артикул товара;
- URL страницы;
- стоимость;
- производитель;
- тип;
- категория;
- заголовок (наименование товара);
- цвета:
 - название цвета:
  - URL-ссылка на изображение;
  - словарь с параметрами для этого цвета:
   - размеры;
   - остатки на складе по размерам;
   - суммарный остаток на складе для каждого цвета;
   - наценки за большие размеры.
Если нет вариантов расцветок, то собираются данные о размерах:
- словарь размеров:
 - размеры;
 - общий остаток на складе;
 - остаток на складе для каждого размера;
 - наценки для больших размеров.
- описание;
- характеристики.
